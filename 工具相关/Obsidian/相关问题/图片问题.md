在ubuntu系统上，使用obsidian粘贴图片，无法粘贴或引用成功，主要问题有两种情况：

# 情况1：复制闪退

**适用情况**：在文件资源管理器中复制图片。

**具体表现**：obsidian直接闪退，图片无法复制成功

# 情况2：无法显示

**适用情况**：
1. 在其他网站上复制图片（deepseek、QQ）
2. 新增情况：在windows系统上图片显示成功，但是在linux电脑上无法显示成功。

**具体表现**：将图片粘贴后，成功出现在路径`/assets/FileName/...png`下，用文件资源管理器可以查看图片，但是在obsidian上无法查看图片

## 解决措施：

**参考来源**： [ubuntu22.04LTS 下附件文件夹内图片无法渲染](https://forum-zh.obsidian.md/t/topic/19362)

快速修复：使用 **deb 包**而不是 snap 版本，图片可以正常在非英语路径下加载了

操作流程：
- commit已经修改的文件
- 删除snap安装的obsidian[snap](../../../Ubuntu/软件的安装与卸载/Ubuntu安装方式/snap.md)
- 下载.deb安装包，重新安装obsidian

**结论**：解决图片显示问题，顺带解决粘贴闪退问题

# 情况3:Mac上文件名问题

**具体介绍**：在macOS系统上，以同样的配置处理插件后，出现了两大问题：
1. 无法粘贴图片，图片不会在新的文件夹下面生成，右上角报错
2. 文件名修改后，无法更新链接

**查看报错**：
- 在[一些进阶操作](../一些进阶操作.md)中，打开obsidian控制台查看报错
- `Generated attachment file name "assets/一些进阶操作。/" is invalid. File name is empty`
- DeepSeek建议，使用简单的命名规则
- 原先命名规则为：`assets/${noteFileName}/${generatedAttachmentFileName}`
- 在[插件仓库中](https://github.com/mnaoumov/obsidian-custom-attachment-location?tab=readme-ov-file#generated-attachment-file-name),采用作者建议的一些命名方式`${date:{momentJsFormat:'YYYYMMDDHHmmssSSS'}}`

**总结**：采用其他命名规则即可解决问题
