# 1. 主要概念

**容器**：容器是一个隔离的docker环境，共享宿主机内核，只隔离进程、文件系统、网络等资源；和虚拟机不同的是，不需要单独的硬件服务器。

**镜像**：每一个镜像是最基本的容器，相当于是一个容器模板，确定了运行环境和安装依赖，就像一个软件的发行版本，镜像可以被不同用户下载使用。

**镜像仓库**：镜像仓库是将不同的镜像存放在一起，官方的镜像仓库是DockHub，用户可以在这上面下载镜像。

# 2.下载安装

基于Linux容器技术，最好的方法就是用Linux系统作为宿主机，Windows和Mac本质都是虚拟化一个Linux虚拟机。

# Docker作用

在Active-call安装过程中，需要通过docker进行安装，故而我有一个疑问：为什么安装active-call镜像需要docker? 不像其他软件或者应用通过安装包或者是curl命令进行安装？以及为什么安装完之后就要把容器删除？

## 传统安装方式的弊端

```text
# 假如用传统方式安装active-call，需要：
1. 安装Python 3.9+
2. 安装CUDA驱动（如果要用GPU）
3. 安装ffmpeg（处理音频）
4. 安装PyTorch
5. 安装各种依赖库
6. 配置环境变量
7. 处理版本冲突
# ... 可能折腾半天还是失败
```

```text
# active-call镜像内部已经包含：
- Ubuntu 22.04（基础系统）
- Python 3.10
- CUDA 11.8
- PyTorch 2.0
- ffmpeg
- 所有依赖库
```

可以发现，docker部署的好处在于：
- 排除了电脑环境不兼容的问题
- 避免了在其他电脑上项目无法运行的困难
- 简化部署流程
