## 方式 1：使用 Jekyll SCSS 编译
```text
scss/          →  编译  →  assets/css/main.css
(scss 源文件)      Jekyll     (编译后输出)
```
  
  **特点**：
  - SCSS 文件存放在 `scss/` 目录（不在 `assets/` 下）
  - Jekyll 会自动编译成 CSS，输出到 `_site/assets/css/`
  - 支持使用变量、mixin、嵌套等 SCSS 特性
  - 适合大型项目，需要复用样式片段

## 方式 2：直接使用 CSS

  ```text
  assets/css/main.css  →  直接复制  →  _site/assets/css/main.css
  ```
  特点：
  - 直接在 assets/ 下编写 .css 文件
  - 无需编译，Jekyll 直接复制到输出目录
  - 适合小型项目或不需要 SCSS 特性的场景

## 关键区别

| 对比项   | SCSS 方式             | 直接 CSS   |
| ----- | ------------------- | -------- |
| **样式复用**  | 支持 @import、mixin、变量 | 无        |
| **文件组织**  | 可拆分为多个模块再编译         | 单文件或手动拆分 |
| **构建流程**  | 需要编译步骤              | 即写即用     |
