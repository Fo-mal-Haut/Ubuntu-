**1. 第一步：定位可执行文件 (`which`)**  
这是所有侦查的起点，它告诉你当你输入命令时，系统实际运行的是哪个文件。

```Bash
which 软件名
# 例如：which firefox, which code
```

**输出示例与解读**：
`/snap/bin/firefox` → **Snap安装**。
`/usr/bin/firefox` → 可能是**APT安装**或**系统自带**，使用包管理器进行判断。
`~/.local/share/flatpak/exports/bin/...` → **Flatpak安装**。
`/opt/vscode/code` → 可能是官方**下载的.tar.gz二进制包**，手动放置。
 **无输出** → 该命令可能不存在，或是一个Shell别名/函数（用 `type 软件名` 进一步检查）。

**2. 第二步：查询各大包管理器记录**  
知道了安装路径，就可以使用对应的**包管理器**。

- **查询APT/dpkg（系统原生包）**：
```bash
# 查看是否通过apt安装及详细信息
dpkg -l | grep -i 软件名
# 更精确查询
dpkg -S $(which 软件名) 2>/dev/null
# 如果知道确切包名，查询该包安装的所有文件
dpkg -L 精确包名
```
- **查询Snap包**：
```bash
snap list | grep -i 软件名
# 查看Snap应用的详细信息
snap info 软件名
```
- **查询Flatpak包**：
```bash
flatpak list | grep -i 软件名
# 或显示所有已安装应用及运行时
flatpak list --app
```
