# 删除废弃包
```Bash
正在搜索废弃的软件 正在读取状态信息... 完成 
删除陈旧的软件包？ 
将删除 652 个软件包。 
包的删除可能会耗费几小时的时间。
```
## **为什么有那么多“废弃软件包”需要删除？**

1. **依赖关系变更**  
    新版本中的软件包可能：
    
    - 被重命名（如 `python2` → `python3`）
        
    - 被拆分或合并
        
    - 被新包替代（如 `gdm` → `gdm3` → 又改回 `gdm`）  
        旧包不再被需要，成为“孤儿”。
        
2. **上游软件淘汰**  
    某些软件在新版本中被移出官方仓库（例如不再维护的旧库、过渡性工具）。
    
3. **架构或初始化系统变更**  
    例如从 `sysvinit` 切换到 `systemd` 时，大量旧初始化脚本包被废弃。
    
4. **元包（metapackage）更新**  
    桌面环境元包（如 `ubuntu-desktop`）在新版本中依赖的软件列表变化，导致旧依赖被标记为废弃。
    
5. **临时性过渡包**  
    上次升级时为了平滑迁移引入的过渡包，在新版本中已无用。
