```Shell
配置文件 '/etc/fwupd/fwupd.conf'
==> 系统中的这个文件或者是由您创建的，或者是由脚本建立的。
==> 软件包维护者所提供的软件包中也包含了该文件。

您现在希望如何处理呢？您有以下几个选择：
Y 或 I ：安装软件包维护者所提供的版本
N 或 O ：保留您原来安装的版本
D ：显示两者的区别
Z ：把当前进程切换到后台，然后查看现在的具体情况

默认的处理方法是保留您当前使用的版本。
*** fwupd.conf (Y/I/N/O/D/Z) [默认选项=N] ?
```

- **这是 dpkg 的子进程**：只响应特定按键
- **输入直接传给 dpkg**：无需按回车（有些版本需要按回车）

# `dpkg` 的**内部选项**：

| 选项        | 实际对应操作                                | 类似命令                      |
| --------- | ------------------------------------- | ------------------------- |
| **Y / I** | `dpkg` 会用新文件覆盖旧文件                     | 类似 `cp 新文件 旧文件`           |
| **N / O** | `dpkg` 保留旧文件，丢弃新文件                    | 类似 `rm 新文件`（不复制）          |
| **D**     | 调用 [diff命令](Linux命令大全/diff命令.md) 比较并显示 | 执行 `diff 旧文件 新文件 \| less` |
| **Z**     | 将 `dpkg` 进程挂起到后台                      | 类似按 `Ctrl+Z` 的效果          |

#  **Z（切换到后台）**：

1. `dpkg` 进程会挂起（暂停）

2. 回到 **shell 提示符**

3. 可以执行任何命令，比如：
```bash
cat /etc/fwupd/fwupd.conf        # 查看当前文件
ls -la /etc/fwupd/                # 查看目录
```
要返回 `dpkg`，输入：
```bash
fg  # 把后台进程带回前台
```
