# 无法访问403

**报错信息**：
```text
git push origin master remote: Write access to repository not granted. 
fatal: 无法访问 'https://github.com/XXXX/XXXX.git/'：The requested URL returned error: 403
```
### 1. **没有写入权限（最常见）**
- 正在推送一个不属于当前账号的仓库
- 或者是fork的仓库但未设置权限
[[Github密钥权限]]
### 2. **认证问题**
- GitHub token/密码过期
- 使用了错误的凭据
### 3. **仓库访问限制**
- 仓库是私有的且未授权
- 组织仓库有访问限制

# 不接受密码认证

**报错信息**：
```text
Pushing to https://github.com/Formula-Haut/Ubuntu-git
remote: Invalid username or token. Password authentication is not supported for Git operations.
fatal: Authentication failed for 'https://github.com/Formula-Haut/Ubuntu-git/'
```

**情况解释**：自2021年8月13日以后，以用户名+密码的方式将不被支持。现在可以支持的方式有： OAuth、SSH Key或者GitHub App installation token) 。

**消息来源**：https://github.blog/changelog/2021-08-12-git-password-authentication-is-shutting-down/

### 方法1：使用 SSH
**参考来源**：[解决github提交代码Support for password authentication was removed 问题](https://cloud.tencent.com/developer/article/1865554)

ssh key的思路如下：在本地，通过sshkeygan的方式，产生一组用于加密的RSA公私钥，之后，git在提交代码的过程中，通过本地的私钥加密，之后将加密数据传递到github服务器。 
而对于github服务端，我们则需要将刚产生的公钥配置在github中，用于对数据的解密。这样就实现了ssh的方式提交代码。
#### **步骤1：检查现有 SSH 密钥**
```bash
ls -al ~/.ssh
# 查看是否有 id_ed25519 或 id_rsa 文件
```
#### **步骤2：生成新的 SSH 密钥**
```bash
# 生成 ED25519 密钥（更安全）
ssh-keygen -t ed25519 -C "你的邮箱@example.com"

# 或生成 RSA 密钥（兼容性更好）
ssh-keygen -t rsa -b 4096 -C "你的邮箱@example.com"
```
产生的公私钥文件将放置在C:\Users\Administrator.ssh目录。如果在linux系统，这个位置就在用户所在的根目录中。 通过cat可以查看：

#### **步骤3：添加 SSH 密钥到 GitHub**

bash

# 复制公钥内容
cat ~/.ssh/id_ed25519.pub
# 或
cat ~/.ssh/id_rsa.pub

**在 GitHub 上添加**：

1. 登录 GitHub → Settings → SSH and GPG keys
    
2. 点击 "New SSH key"
    
3. 粘贴公钥内容
    
4. 点击 "Add SSH key"
    

#### **步骤4：修改远程地址为 SSH**

bash

# 查看当前远程地址
git remote -v

# 修改为 SSH 地址
git remote set-url origin git@github.com:Formula-Haut/Ubuntu-git.git

# 验证连接
ssh -T git@github.com
# 应该看到：Hi Formula-Haut! You've successfully authenticated...
